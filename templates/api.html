API


GET      /<apikey>

Get the current state of the game in JSON format

Parameters

apikey: Your personal API key

Return value

The game state in JSON format

Example response

POST      /<apikey>/fold

Fold the current hand

Parameters

apikey: Your personal API key

Return value

The game state in JSON format with the additional parameter posted: true

Example response

POST      /<apikey>/check

Check the current hand

(Only applies if you have already matched the current bet)

Parameters

apikey: Your personal API key

Return value

The game state in JSON format with the additional parameter posted: true

Example response

POST      /<apikey>/call

Call the current bet

Parameters

apikey: Your personal API key

Return value

The game state in JSON format with the additional parameter posted: true

Example response

POST      /<apikey>/bet/<amount>

Bet an amount of money OVER the current call amount

Parameters

apikey: Your personal API key

amount: The amount to bet in excess of the current call amount

Return value

The game state in JSON format with the additional parameter posted: true

Example response

Example JSON Response 

{
    "round_id": 1,
    "initial_stack": 1000,
    "stack": 700,
    "current_bet": 375,
    "call_amount": 75,
    "current_pot": 975,
    "phase": PHASE_RIVER,
    "my_turn": false,
    "hand": ["AH", "QH"],
    "community": ["KH", "JD", "4C", "AC", "QC"],
    "opponents": [
        {
            "name": "Ralph",
            "initial_stack": 1000,
            "stack": 700,
            "folded": false
        },
        {
            "name": "Steve",
            "initial_stack": 1000,
            "stack": 625,
            "folded": false
        },
        {
            "name": "Jack",
            "intial_stack": 1000,
            "stack": 1000,
            "folded": true
        }
    ],
    "posted": true
}

Explanation

round_id: an ascending number used to represent which "hand" is currently being played

initial_stack: how many chips the player had at the beginning of the round

stack: how many chips the player currently has

current_bet: what the current bet is for each player at the table

call_amount: the amount required to call the current bet

current_pot: total amount of chips currently in the pot

phase: one of PHASE_PREFLOP, PHASE_FLOP, PHASE_TURN, or PHASE_RIVER

my_turn: a boolean representing whether or not it is your turn to act

hand: an array of 0-2 cards representing your two hole cards

community: an array of 0-5 cards representing the community cards

opponents: an array of all opponents currently at your table

posted: only sent after a POST; set to true to indicate success

Card Format

Cards will be given as a string of two characters. The first character represents the RANK of the card. This will be noted by a number 2-9, T (for ten), J (for Jack), Q (for Queen), K (for King), or A (for Ace). The second character represents the SUIT of the card. This will be noted by C (for Clubs), D (for Diamonds), S (for Spades), or H (for Hearts).

In other words, each card conforms to the regular expression [2-9TJQKA][CDSH].
